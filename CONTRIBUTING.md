# Contributing to med-scheduler

Thanks for your interest in contributing! This project helps doctors create medication reminders for their patients.

## How to contribute

1. **Fork** the repository
2. **Clone** your fork: `git clone https://github.com/your-username/med-scheduler.git`
3. **Create a branch**: `git checkout -b feature/your-feature-name`
4. **Make your changes** and test them locally
5. **Commit** with a clear message: `git commit -m "Add: brief description"`
6. **Push** to your fork: `git push origin feature/your-feature-name`
7. **Open a Pull Request** against `main`

## Development setup

Just open `frontend/index.html` in your browser. No build tools required.

## Code conventions

- **JavaScript**: Use `const`/`let` (no `var`). Standard semicolons.
- **CSS**: Use the existing CSS custom properties (variables) defined in `style.css`.
- **i18n**: Add translations for both `es` and `en` in `i18n.js` for any new UI text.

## Reporting bugs

Open an issue with:
- Steps to reproduce
- Expected vs actual behavior
- Browser/OS if relevant

## Feature ideas

We welcome ideas! Open an issue tagged `enhancement` and describe:
- The problem you're trying to solve
- Your proposed solution

## Code of Conduct

- Be respectful and constructive
- Welcome newcomers
- Focus on the work, not the person

Thank you for helping make med-scheduler better!
